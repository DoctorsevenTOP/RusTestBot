# Mini Apps для тестов на вставку букв

## Обзор

Этот проект включает два новых Telegram Mini Apps для работы с тестами на вставку букв:

1. **Insert Test App** (`insert_test_app.html`) - интерфейс для прохождения тестов
2. **Test Builder App** (`test_builder_app.html`) - интерфейс для создания тестов

## Команды бота

### Для всех пользователей:

- `/insert_test` - Запустить новый Mini App для прохождения тестов на вставку букв
- `/insert` - Старый интерфейс (для обратной совместимости)

### Для администраторов:

- `/insert_builder` - Запустить Mini App для создания тестов на вставку букв
- `/edit_test` - Старый редактор тестов (для обратной совместимости)

## Insert Test App - Прохождение тестов

### Функциональность:

✅ **Список доступных тестов**
- Показывает все активные тесты с вопросами типа `insert_letter`
- Отображение названия, описания и количества вопросов
- Выбор теста для прохождения

✅ **Интерактивный интерфейс**
- Нажатие на пропуск активирует его для ввода
- Панель с буквами для выбора
- Визуальная обратная связь (цветовая индикация)
- Тактильная отдача (HapticFeedback)

✅ **Проверка ответов**
- Мгновенная проверка правильности
- Показ правильных ответов для неправильных
- Подсчет процентовного соотношения

✅ **Экран результатов**
- Итоговый счет и процент
- Детализация по каждому вопросу
- Отправка результатов боту
- Возможность пройти тест заново

### Как использовать:

1. Отправьте боту `/insert_test`
2. Выберите тест из списка
3. Нажмите "Начать тест"
4. Нажимайте на пропуски и выбирайте буквы
5. Нажмите "Проверить ответы"
6. Просмотрите результаты и отправьте их боту

### Структура данных теста:

```json
{
  "test_id": "insert_1",
  "title": "Правописание -Н- и -НН-",
  "description": "Вставьте пропущенные буквы",
  "questions": [
    {
      "type": "insert_letter",
      "text": "В комнате стоял див..н, который был украше.. узором.",
      "gaps": [
        {"position": 0, "correct": "а"},
        {"position": 1, "correct": "н"}
      ],
      "letters": ["а", "о", "н", "е"]
    }
  ]
}
```

## Test Builder App - Создание тестов

### Функциональность:

✅ **Основная информация**
- Название теста
- Описание теста

✅ **Конструктор вопросов**
- Ввод текста с пропусками (используйте `..`)
- Автоматическое обнаружение пропусков
- Настройка правильных букв для каждого пропуска
- Добавление вариантов букв для выбора

✅ **Управление вопросами**
- Добавление неограниченного количества вопросов
- Предпросмотр вопросов с заполненными пропусками
- Удаление вопросов
- Статистика (количество вопросов, пропусков, букв)

✅ **Действия**
- Сохранение теста локально
- Предпросмотр теста (открывает Insert Test App с текущими данными)
- Отправка в бот (создание теста в базе данных)

### Как использовать:

1. Отправьте боту `/insert_builder` (только для администраторов)
2. Введите название и описание теста
3. Нажмите "Добавить" чтобы создать вопрос
4. Введите текст с пропусками, используя `..` для пропусков
5. Укажите правильные буквы для каждого пропуска
6. Добавьте варианты букв (минимум 2)
7. Нажмите "Добавить вопрос"
8. Повторите для всех вопросов
9. Нажмите "Отправить в бот" для сохранения

### Пример создания вопроса:

**Текст с пропусками:**
```
В комнате стоял див..н, который был украше.. узором.
```

**Правильные буквы:**
- Пропуск 1: а
- Пропуск 2: н

**Варианты букв:**
- а, о, н, е

### Структура отправляемых данных:

```json
{
  "action": "create_insert_test",
  "title": "Правописание -Н- и -НН-",
  "description": "Вставьте пропущенные буквы в слова",
  "active": true,
  "questions": [
    {
      "type": "insert_letter",
      "text": "В комнате стоял див..н, который был украше.. узором.",
      "gaps": [
        {"position": 0, "correct": "а"},
        {"position": 1, "correct": "н"}
      ],
      "letters": ["а", "о", "н", "е"],
      "description": "Вставьте пропущенные буквы"
    }
  ]
}
```

## Интеграция с ботом

### Обработка данных

Бот обрабатывает следующие типы данных из Mini Apps:

1. **`get_insert_tests`** - Запрос списка тестов
2. **`create_insert_test`** - Создание нового теста
3. **`submit_insert_test`** - Отправка результатов теста

### Сохранение результатов

Результаты тестов сохраняются в базе данных и включают:
- ID пользователя
- ID теста
- Количество правильных ответов
- Общее количество вопросов
- Процентное соотношение
- Детализация ответов

### Таблица лидеров

Результаты отображаются в таблице лидеров, доступной через кнопку "🏆 Таблица лидеров" в главном меню.

## Развёртывание

### Требования

1. Веб-хостинг с поддержкой HTTPS (GitHub Pages, Vercel, Netlify и т.д.)
2. Telegram Bot Token
3. Настроенный вебхук или polling

### Шаги развёртывания

1. **Загрузите HTML файлы на хостинг:**
   ```
   https://your-domain.com/insert_test_app.html
   https://your-domain.com/test_builder_app.html
   ```

2. **Обновите URL в main.py:**
   
   В команде `/insert_test`:
   ```python
   app_url = "https://your-domain.com/insert_test_app.html"
   ```
   
   В команде `/insert_builder`:
   ```python
   app_url = "https://your-domain.com/test_builder_app.html"
   ```

3. **Настройте Mini App в BotFather:**
   - Отправьте `/mybots` @BotFather
   - Выберите вашего бота
   - Bot Settings → Menu Button
   - Установите URL и текст кнопки

4. **Запустите бота:**
   ```bash
   python main.py
   ```

### Локальное тестирование

Для локального тестирования без хостинга:

1. Откройте HTML файл в браузере
2. Добавьте параметр `test_data` в URL для предзаполнения тестов:
   ```
   insert_test_app.html?test_data={"tests":[...]}
   ```

## Особенности реализации

### Telegram WebApp API

Используемые функции:
- `tg.ready()` - Инициализация
- `tg.expand()` - Разворот на весь экран
- `tg.sendData()` - Отправка данных боту
- `tg.close()` - Закрытие Mini App
- `tg.HapticFeedback` - Тактильная отдача
- `tg.initDataUnsafe` - Данные о пользователе

### Совместимость

- Работает на iOS и Android
- Адаптивный дизайн
- Поддержка тем Telegram (использует CSS переменные)
- Оптимизировано для мобильных устройств

## Устранение проблем

### Mini App не открывается

**Причина:** Неверный URL или проблемы с хостингом
**Решение:** Проверьте что HTML файлы доступны по HTTPS ссылке

### Данные не отправляются боту

**Причина:** Проблемы с обработчиком web_app_data
**Решение:** Проверьте логи бота, убедитесь что JSON валидный

### Тесты не отображаются

**Причина:** Нет тестов с вопросами типа `insert_letter`
**Решение:** Создайте тест через `/insert_builder` или `/load_test`
