# Telegram бот для проверки знаний по русскому языку

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Получите токен бота:
   - Откройте @BotFather в Telegram
   - Создайте нового бота командой `/newbot`
   - Скопируйте полученный токен

3. Настройте `config.py`:
   - Замените токен на ваш
   - `ADMIN_IDS` можно оставить пустым `[]` — администраторы управляются через SQLite

## Запуск

```bash
python main.py
```

## Развёртывание на сервере

### Быстрый старт с Docker:

```bash
# 1. Клонировать репозиторий
git clone <repo>
cd TgRusCheck

# 2. Настроить config.py
nano config.py

# 3. Запустить через docker-compose
docker-compose up -d

# 4. Назначить администратора ВНУТРИ контейнера
docker-compose exec bot python set_admin.py <telegram_id>

# 5. Проверить работу
docker-compose logs -f
```

### Миграция данных (если используете старую версию с JSON):

```bash
# Запустить миграцию внутри контейнера
docker-compose exec bot python migrate.py

# После успешной миграции можно удалить старые файлы
docker-compose exec bot rm data/*.json
```

### Проверка прав доступа:
```bash
# Проверить владельца данных
ls -la data/

# Исправить права (если нужно)
chmod 777 data/
```

### Команды Docker:
```bash
# Просмотр логов
docker-compose logs -f

# Перезапуск
docker-compose restart

# Остановка
docker-compose down

# Войти в контейнер для отладки
docker-compose exec bot bash

# Проверка базы данных
docker-compose exec bot sqlite3 /app/data/bot.db ".tables"
```

### Отладка:
Отправьте боту команду `/ping` — он покажет:
- ✅ Работает ли бот
- 📁 Путь к базе данных
- ✅ Права на запись

### Назначение администратора:

**Важно:** Запускайте команду **внутри контейнера**, а не на хосте!

```bash
# Через docker-compose (рекомендуется)
docker-compose exec bot python set_admin.py <telegram_id>

# Или через docker
docker exec <container_name> python set_admin.py <telegram_id>

# Пример:
docker-compose exec bot python set_admin.py 5153839634
```

**Проверка:**
```bash
# Проверить базу данных
docker-compose exec bot sqlite3 /app/data/bot.db "SELECT * FROM users;"
```

## Функционал

### Для всех пользователей:
- `/start` - начало работы, запрос имени, отображение Telegram ID
- `/help` - справка по всем командам
- 📋 **Список тестов** - просмотр доступных тестов и выбор через кнопки
- 🏆 **Таблица лидеров** - лучшие результаты по каждому тесту
- 📊 **Мои результаты** - история пройденных тестов

### Для администраторов:

#### Способ 1: Интерактивное создание в боте
- ➕ **Создать тест** - пошаговое создание теста
  - Ввод названия
  - Ввод описания
  - Добавление вопросов с правильными ответами
  - Завершение создания

#### Способ 2: Загрузка JSON (рекомендуется)
1. Откройте `test_builder.html` в браузере
2. Заполните название, описание и вопросы
3. Нажмите "Сгенерировать JSON"
4. Скопируйте результат
5. Отправьте боту JSON текст или файл
6. Или используйте команду `/load_test` для инструкции

### Команды администратора:
- `/load_test` - инструкция по загрузке JSON
- `/toggle_test` - включить/выключить тесты
- `/set_admin` - назначить/снять права администратора
- `/cancel` - отмена текущего действия

### Управление активностью тестов:
1. `/toggle_test` - показать список всех тестов со статусами
2. `/toggle_test <ID>` - переключить статус теста (активен/отключён)

**Статусы:**
- 🟢 **Активен** - тест виден пользователям, можно проходить
- 🔴 **Отключён** - тест скрыт из списка, нельзя пройти

### Управление правами администратора:
Права администратора хранятся в `data/users.json` в поле `admin: true/false`.

**Команда `/set_admin`:**
```
/set_admin <user_id> <0|1>
```

**Примеры:**
```
/set_admin 123456789 1  # Назначить администратором
/set_admin 123456789 0  # Снять права администратора
```

**Ручное редактирование users.json:**
```json
{
  "123456789": {
    "name": "Иван",
    "telegram_id": 123456789,
    "admin": true,
    "registered_at": "2026-03-23T..."
  }
}
```

**Примечание:** Только текущие администраторы могут использовать `/set_admin`. Первый администратор указывается в `config.py` (ADMIN_IDS).

## Структура данных

Данные хранятся в папке `data/`:
- `users.json` - пользователи (имя, ID, дата регистрации)
- `tests.json` - тесты (название, описание, вопросы, ответы)
- `results.json` - результаты прохождений

## Формат JSON для загрузки

### Одиночный ответ:
```json
{
  "title": "Название теста",
  "description": "Описание теста",
  "active": true,
  "questions": [
    {
      "type": "single",
      "text": "Вопрос 1",
      "answer": "Правильный ответ"
    }
  ]
}
```

### Множественный выбор:
```json
{
  "title": "Название теста",
  "description": "Описание теста",
  "active": true,
  "questions": [
    {
      "type": "multiple",
      "text": "Выберите все глаголы:",
      "options": ["бежать", "красивый", "думать", "синий"],
      "answer": ["бежать", "думать"]
    }
  ]
}
```

**Поля теста:**
- `title` — название теста
- `description` — описание
- `active` — статус активности (true/false)
- `questions` — массив вопросов

**Поля вопроса:**
- `type` — тип вопроса: `"single"` (одиночный) или `"multiple"` (множественный)
- `text` — текст вопроса
- `answer` — правильный ответ (строка для single, массив для multiple)
- `options` — варианты ответов (только для multiple)

## Прохождение теста

1. Выберите "📋 Список тестов"
2. Нажмите на кнопку с названием теста
3. Нажмите "▶️ Начать тест"
4. Вводите ответы на вопросы
5. Получите результат в конце
